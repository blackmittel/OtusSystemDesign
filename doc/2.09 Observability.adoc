=== Observability

==== Показатели

===== Бизнес-процессы
    *   Количество активных пользователей, сессий
    *   Создание/просмотры объявлений
    *   Создание/оплата/доставка заказов
    *   Успешность платежей, средний чек
    *   Скорость работы поиска, релевантность результатов
===== Техническое состояние
    *   Доступность всех сервисов (API Gateway, авторизация, поиск, заказы, платежи)
    *   Время ответа каждого сервиса
    *   Загрузка процессора, памяти, дисков на всех серверах
    *   Состояние баз данных (PostgreSQL, Elasticsearch, Redis)
    *   Очереди сообщений в RabbitMQ (нет накопления)
    *   Работа внешних API (платежные системы, службы доставки)

==== Средства

===== Дашборды (Grafana)
    * *"Общее состояние"*: Сводка по всем критичным сервисам, общее количество заказов, ошибок.
    * *"Финансы"*: Статусы платежей, успешные/неуспешные транзакции, среднее время проведения.
    * *"Поиск и каталог"*: Время ответа поиска, количество запросов, загрузка Elasticsearch.
    * *"Инфраструктура"*: Загрузка серверов, состояние баз данных, использование кэша.

===== Автоматическое оповещение
    * Когда любой сервис недоступен более 1 минуты
    * При росте количества ошибок платежей выше 5%
    * Если время ответа поиска превышает 2 секунды
    * При заполнении диска на сервере БД более 85%
    * При сбоях в работе брокера сообщений или кэша
    * Все алерты поступают в единый Telegram-чат с приоритетами (критический/предупреждение)

==== Инструменты диагностики
    * *Логи (Kibana)*: Логи из всех сервисов хранятся централизованно.
    * *Трассировка (Jaeger)*: Путь выполнения запросов.
    * *Инфраструктурные метрики (Prometheus)*: Детальные показатели по каждому серверу и базе данных в реальном времени.
