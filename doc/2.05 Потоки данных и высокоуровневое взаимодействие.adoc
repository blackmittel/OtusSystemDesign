=== Потоки данных и высокоуровневое взаимодействие

==== Информационные потоки

[cols="1,2,3,1,3",options="header"]
|===
| ID | Группа | Участники | Тип | Описание

| IF-01
| Инфраструктура
| Client → GeoDNS → API Gateway
| Sync
| Геомаршрутизация запроса в ЦОД (Москва / Краснодар)

| IF-02
| Инфраструктура
| Client → CDN
| Sync
| Получение медиаконтента из ближайшего узла CDN (СПб, Казань)

| IF-03
| Инфраструктура
| API Gateway → Redis
| Sync
| Кэширование ответов; снятие нагрузки с бизнес-сервисов

| IF-04
| Безопасность
| API Gateway → Auth Service
| Sync
| Проверка токена / валидация сессии на каждый входящий запрос

| IF-05
| Пользователи
| Client → API Gateway → User Service
| Sync
| Управление профилем, wish-листом; чтение поведенческих данных

| IF-06
| Объявления
| Client → API Gateway → Ticket Service
| Sync
| CRUD объявлений; модерация; резервирование товара при создании заказа

| IF-07
| Объявления
| Client → API Gateway → Media Service
| Sync
| Загрузка медиафайлов (фото, видео товара) при создании / редактировании объявления

| IF-08
| Объявления
| Media Service → CDN
| Sync
| Публикация загруженных медиафайлов в CDN для раздачи

| IF-09
| Поиск
| Client → API Gateway → Search Service
| Sync
| Параметрический и текстовый поиск объявлений; персональные рекомендации

| IF-10
| Поиск
| Kafka → Search Service
| Async
| Индексация новых и изменённых объявлений; поддержание актуальности поискового индекса

| IF-11
| Сообщения
| Client → API Gateway → Ticket Service
| Sync (polling)
| Обмен сообщениями между покупателем и продавцом — до оформления заказа и в его контексте

| IF-12
| Заказ
| Client → API Gateway → Order Service
| Sync
| Создание заказа покупателем

| IF-13
| Заказ
| Order Service → Ticket Service
| Sync
| Валидация объявления и резервирование товара в момент оформления заказа

| IF-14
| Заказ
| Order Service → Delivery Service
| Sync
| Расчёт стоимости и вариантов доставки при оформлении заказа

| IF-15
| Заказ
| Delivery Service → Ext. Logistics API
| Sync
| Запрос тарифов и оформление отправления у логистического провайдера

| IF-16
| Платежи
| Client → API Gateway → Payment Service
| Sync
| Инициация платежа покупателем

| IF-17
| Платежи
| Order Service → Payment Service
| Sync
| Привязка созданного платежа к конкретному заказу

| IF-18
| Платежи
| Ext. Payment API → Payment Service
| Sync (webhook)
| Подтверждение / отклонение платежа от внешнего платёжного шлюза

| IF-19
| Жизненный цикл заказа
| Payment Service → Kafka → Order Service
| Async
| Событие подтверждения платежа; Order Service переводит заказ в статус «оплачен»

| IF-20
| Жизненный цикл заказа
| Delivery Service → Kafka → Order Service
| Async
| Обновление статуса доставки (передан в логистику, в пути, доставлен)

| IF-21
| Расчёты и выплаты
| Kafka → Billing Service
| Async
| Данные завершённой сделки для расчёта комиссии торговой площадки

| IF-22
| Расчёты и выплаты
| Billing Service → Payment Service
| Sync
| Инициация выплаты продавцу после завершения сделки и удержания комиссии

| IF-23
| Аналитика и уведомления
| Kafka → Analytics Service
| Async
| Агрегация событий заказов, платежей и пользовательского поведения

| IF-24
| Аналитика и уведомления
| Analytics Service → Client (push / email / SMS)
| Async
| Отправка уведомлений пользователям (статус заказа, рекомендации, акции)

| IF-25
| Поведенческие данные
| Kafka → User Service
| Async
| Обновление поведенческого профиля пользователя на основе его активности в системе

| IF-26
| Аналитика (ClickHouse)
| Kafka → ClickHouse
| Async
| Инжест всех событий (user_events, ads_events, orders, payments, search) в ClickHouse через Kafka Engine и materialized views

| IF-27
| Аналитика (ClickHouse)
| ClickHouse → Grafana
| Sync
| Построение BI-дашбордов (DAU, MAU, CTR, GMV, revenue) и визуализация метрик в реальном времени

| IF-28
| Аналитика (ClickHouse)
| ClickHouse → Recommendation Service
| Sync / Async
| Предоставление агрегатов (популярность, тренды, co-view/co-click) для формирования рекомендаций

| IF-29
| Аналитика (ClickHouse)
| ClickHouse → Search Service
| Async
| Обогащение поискового индекса сигналами ранжирования (популярность, конверсии, freshness)

| IF-30
| Аналитика (ClickHouse)
| ClickHouse → Kafka
| Async
| Публикация агрегированных событий (триггеры: популярное объявление, аномалии, всплески активности)

| IF-31
| Аналитика (ClickHouse)
| ClickHouse → Notification Service
| Async
| Генерация событий для пользовательских уведомлений (рост интереса к объявлению, рекомендации, изменения цен)

| IF-32
| Аналитика (ClickHouse)
| ClickHouse → Anti-Fraud Service
| Sync / Async
| Передача агрегированных поведенческих паттернов для выявления аномалий и мошенничества

| IF-33
| Аналитика (ClickHouse)
| Search Service → Kafka → ClickHouse
| Async
| Логирование поисковых запросов и кликов пользователей для последующего анализа и улучшения ранжирования

| IF-34
| Аналитика (ClickHouse)
| API Gateway → Kafka → ClickHouse
| Async
| Логирование пользовательских действий (clickstream) для построения полной картины поведения

| IF-35
| Аналитика (ClickHouse)
| ClickHouse → Data Export / Backup
| Async (batch)
| Выгрузка агрегированных данных в хранилище или внешние системы для долгосрочного хранения и офлайн-аналитики

|===
