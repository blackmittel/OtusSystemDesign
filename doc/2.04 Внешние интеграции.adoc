=== Внешние интеграции

==== Интеграция с платёжными агрегаторами

* Взаимодействие с платёжными агрегаторами поддерживается через предоставляемые ими API с использованием протокола HTTPS.
* Должны поддерживаться следующие способы оплаты: банковские карты (Visa, Mastercard, МИР), электронные кошельки, СБП
* Система должна получать уведомления о статусе платежа (успешный, отклоненный, ожидает подтверждения) в режиме реального времени через webhook.
* Должна быть реализована возможность возврата средств покупателю через API агрегатора.
* Система должна обеспечивать выплаты продавцам с учетом комиссии маркетплейса.
* Должно обеспечиваться логирование всех транзакций с сохранением идентификаторов платежей в обеих системах.
* При недоступности платежного агрегатора система должна информировать пользователя о временной невозможности совершить платеж.
* Система должна поддерживать механизм повторных попыток при сбоях (retry mechanism) с экспоненциальной задержкой.
* Взаимодействие платежного сервиса с API агрегатора осуществляется в синхронном режиме.

*Передаваемые данные:*

Исходящие: сумма платежа, идентификатор заказа, данные покупателя (email, телефон), URL для возврата после оплаты

Входящие: статус платежа, идентификатор транзакции, timestamp операции, детали ошибки (при наличии)

==== Интеграция с транспортной компанией
Назначение: оформление доставки заказов, отслеживание статуса доставки.

* Система должна автоматически создавать заявку на доставку через API транспортной компании при подтверждении заказа
* Должен рассчитываться предварительный срок и стоимость доставки на основе данных о весе, габаритах товара и адресе получателя
* Система должна получать актуальные данные о тарифах доставки для отображения покупателю при оформлении заказа
* Должна обеспечиваться генерация транспортных накладных и печатных форм для маркировки отправлений
* Система должна получать обновления статуса доставки (принято на склад, в пути, доставлено, возврат) через webhook или polling с интервалом не более 30 минут
* Должна быть реализована возможность вызова курьера для забора товара от продавца
* Система должна поддерживать отмену доставки и оформление возврата через API транспортной компании
* При недоступности API система должна использовать последние загруженные тарифы и предупреждать администратора о необходимости ручной обработки

*Передаваемые данные:*

Исходящие: параметры отправления (вес, габариты, объявленная стоимость), адреса отправителя и получателя, контактные данные, идентификатор заказа в системе маркетплейса

Входящие: трек-номер отправления, статус доставки, история перемещения, стоимость доставки, дата доставки (фактическая/планируемая)
