=== Форматы взаимодействия

[NOTE]
====
Раздел описывает структуру и формат данных при взаимодействии компонентов.
Все форматы соответствуют схемам данных, описанным в разделе «Схемы данных».
Для всех операций, не имеющих явного примера, используются те же принципы
и соответствующие схемы из раздела «Схемы данных».
====

==== DNS / GeoDNS

[cols="1,2",options="header"]
|===
| Формат | Описание

| A-запись (Address Record) | Преобразует доменное имя в IPv4-адрес. Используется для геораспределения трафика между ЦОД (Москва/Краснодар). GeoDNS возвращает IP-адрес ближайшего ЦОД на основе местоположения клиента.

| SRV-запись (Service Record) | Определяет хост и порт для конкретного сервиса. Используется для определения активного API Gateway в каждом ЦОД. Включает приоритет и вес для failover.

|===

==== HTTP/REST (JSON)

Формат для внешних API (Client -> API Gateway).

===== Общие принципы

- Кодировка: UTF-8
- Content-Type: application/json
- Версионирование: через URL path (`/v1/`, `/v2/`)
- Пагинация: cursor-based (after_id, limit) — клиент передаёт `limit` и идентификатор последнего полученного элемента
- Ошибки: формат RFC 9457 (Problem Details for HTTP APIs)

===== Форматы запросов и ответов

Все форматы запросов и ответов соответствуют схемам из раздела «Схемы данных»:

- `User`, `SellerProfile` — для операций с пользователями
- `Ticket`, `Category` — для операций с объявлениями
- `Order`, `Reservation` — для операций с заказами
- `Message`, `Conversation` — для операций с сообщениями
- `Payment`, `Payout`, `Refund`, `Transaction` — для платёжных операций
- `Shipment`, `DeliveryService` — для операций доставки
- `Review`, `Notification` — для отзывов и уведомлений

Поля запросов и ответов соответствуют полям этих схем (см. раздел «Схемы данных»).

===== Пример: создание объявления (POST /v1/tickets)

Запрос (соответствует схемам `Ticket`, `Category`):

[source,json]
----
{
    "title": "string",                   // Ticket.title
    "description": "string",             // Ticket.description
    "category_id": "uuid",               // Ticket.category_id
    "price": 0,                          // Ticket.price
    "total_quantity": 0,                 // Ticket.total_quantity
    "images": ["string"]                 // массив URL изображений
}
----

Ответ (201 Created):

[source,json]
----
{
    "ticket_id": "uuid",                 // Ticket.ticket_id
    "status": "string",                  // Ticket.status
    "created_at": "datetime"             // Ticket.created_at
}
----

===== Пример: создание заказа (POST /v1/orders)

Запрос (соответствует схемам `Order`, `Reservation`):

[source,json]
----
{
    "ticket_id": "uuid",                 // Order.ticket_id
    "quantity": 0,                       // Order.quantity
    "delivery_address": {                // адрес доставки
        "city": "string",
        "street": "string",
        "house": "string"
    }
}
----

Ответ (201 Created):

[source,json]
----
{
    "order_id": "uuid",                  // Order.order_id
    "status": "string",                  // Order.status
    "total_price": 0,                    // Order.total_price
    "payment_url": "string",             // URL для оплаты
    "created_at": "datetime"             // Order.created_at
}
----

===== Пример ответа с ошибкой (4xx/5xx)

Формат RFC 9457:

[source,json]
----
{
    "type": "string",                    // URI типа ошибки
    "title": "string",                   // краткое описание
    "status": 0,                         // HTTP статус
    "detail": "string",                  // детальное описание
    "instance": "string"                 // URI запроса
}
----

===== Остальные операции

Для остальных операций (просмотр профиля, редактирование объявления, поиск, сообщения, получение статуса заказа и т.д.) используются:

- Те же принципы кодирования и версионирования
- Соответствующие схемы из раздела «Схемы данных»
- Поля, указанные в схемах (например, `GET /v1/users/{user_id}` возвращает `User`, `GET /v1/tickets/{ticket_id}` возвращает `Ticket` и т.д.)

==== gRPC (Protobuf)

Формат для внутренних синхронных вызовов между сервисами внутри ЦОД.

Все gRPC-сообщения соответствуют схемам данных:

- `User`, `UserRole`, `Role` — для Auth Service
- `Order`, `Payment` — для Payment Service
- `Order`, `Shipment`, `DeliveryService` — для Delivery Service
- `Payout`, `Transaction` — для Billing Service

===== Auth Service (auth.proto)

[source,protobuf]
----
syntax = "proto3";

service AuthService {
    rpc ValidateToken (ValidateTokenRequest) returns (ValidateTokenResponse);
    rpc Authenticate (AuthenticateRequest) returns (AuthenticateResponse);
}

message ValidateTokenRequest {
    string token = 1;                    // JWT или session token
}

message ValidateTokenResponse {
    bool valid = 1;                      // признак валидности токена
    string user_id = 2;                  // User.user_id
    repeated string roles = 3;           // роли из UserRole + Role
    int64 expires_at = 4;                // timestamp истечения
}

message AuthenticateRequest {
    string login = 1;                    // email или телефон (User.email/phone)
    string password = 2;                 // пароль (User.password_hash)
}

message AuthenticateResponse {
    string session_id = 1;               // идентификатор сессии
    string jwt_token = 2;                // JWT токен
    int64 expires_in = 3;                // время жизни в секундах
}
----

===== Payment Service (payment.proto)

[source,protobuf]
----
syntax = "proto3";

service PaymentService {
    rpc CreatePayment (CreatePaymentRequest) returns (CreatePaymentResponse);
    rpc Payout (PayoutRequest) returns (PayoutResponse);
}

message CreatePaymentRequest {
    string order_id = 1;                 // Order.order_id
    int64 amount = 2;                    // сумма в копейках (Order.total_price)
    string currency = 3;                 // валюта (RUB)
    string return_url = 4;               // URL возврата после оплаты
    string webhook_url = 5;              // URL для уведомлений
}

message CreatePaymentResponse {
    string payment_id = 1;               // Payment.payment_id
    string payment_url = 2;              // URL для редиректа на оплату
    string status = 3;                   // Payment.status
}

message PayoutRequest {
    string seller_id = 1;                // SellerProfile.seller_id
    int64 amount = 2;                    // сумма выплаты
    string currency = 3;                 // валюта
    string payout_id = 4;                // Payout.payout_id
}

message PayoutResponse {
    string payout_id = 1;                // Payout.payout_id
    string status = 2;                   // Payout.status
    string transaction_id = 3;           // Transaction.transaction_id
}
----

===== Delivery Service (delivery.proto)

[source,protobuf]
----
syntax = "proto3";

service DeliveryService {
    rpc CalculateDelivery (DeliveryRequest) returns (DeliveryResponse);
    rpc CreateShipment (ShipmentRequest) returns (ShipmentResponse);
}

message DeliveryRequest {
    string order_id = 1;                 // Order.order_id
    string address = 2;                  // адрес доставки
    int32 quantity = 3;                  // количество товаров (Order.quantity)
}

message DeliveryResponse {
    repeated DeliveryOption options = 1; // варианты доставки
}

message DeliveryOption {
    string delivery_service_id = 1;      // DeliveryService.delivery_service_id
    string name = 2;                     // DeliveryService.name
    int64 price = 3;                     // стоимость доставки
    int32 days = 4;                      // срок в днях
}

message ShipmentRequest {
    string order_id = 1;                 // Order.order_id
    string delivery_service_id = 2;      // DeliveryService.delivery_service_id
    string address = 3;                  // адрес доставки
}

message ShipmentResponse {
    string shipment_id = 1;              // Shipment.shipment_id
    string tracking_number = 2;          // Shipment.tracking_number
    string status = 3;                   // Shipment.status
}
----

==== HTTP Webhook (JSON)

Формат для внешних асинхронных колбэков.

Все webhook-сообщения соответствуют схемам `Payment`, `Transaction`, `Shipment`.

===== Webhook платежей

[source,json]
----
POST /webhook/payment
{
    "payment_id": "uuid",                // Payment.payment_id
    "order_id": "uuid",                  // Order.order_id
    "status": "string",                  // Payment.status (Succeeded, Failed)
    "provider": "string",                // Payment.provider
    "provider_txn_id": "string",         // Transaction.external_reference
    "amount": 0,                         // Payment.amount
    "currency": "string",                // валюта
    "timestamp": "datetime",             // время события
    "signature": "string"                // подпись для верификации
}
----

===== Webhook доставки

[source,json]
----
POST /webhook/delivery
{
    "shipment_id": "uuid",               // Shipment.shipment_id
    "order_id": "uuid",                  // Order.order_id
    "status": "string",                  // Shipment.status
    "tracking_number": "string",         // Shipment.tracking_number
    "carrier": "string",                 // DeliveryService.name
    "timestamp": "datetime"              // время события
}
----

==== Redis / Cache (RESP)

Формат для кэширования.

[cols="2,2,3",options="header"]
|===
| Шаблон ключа | TTL | Значение (JSON)

| `session:{session_id}` | 24h | `{"user_id": "uuid", "roles": ["string"], "expires_at": "datetime"}`

| `token:{jwt}` | 1h | `{"user_id": "uuid", "blocked": false}`

| `search:{query_hash}` | 5m | `{"ticket_ids": ["uuid"], "total": 0}`

| `ticket:{ticket_id}` | 10m | полный объект Ticket (см. схему)

| `user:{user_id}` | 10m | полный объект User (см. схему)

|===

==== S3 / CDN

Формат для медиафайлов.

===== Схема URL

[source]
----
https://cdn.{city}.example.com/media/{ticket_id}/{file_id}/{version}.{ext}
----

где:
- `{city}` — spb или kzn (ближайший CDN узел)
- `{ticket_id}` — Ticket.ticket_id
- `{file_id}` — уникальный идентификатор файла
- `{version}` — версия файла (для инвалидации кэша)
- `{ext}` — расширение файла

===== Метаданные (S3)

[source,json]
----
{
    "ticket_id": "uuid",                 // Ticket.ticket_id
    "user_id": "uuid",                   // Ticket.seller_id
    "content_type": "string",            // MIME тип
    "size_bytes": 0,                     // размер в байтах
    "width": 0,                          // ширина изображения
    "height": 0,                         // высота изображения
    "created_at": "datetime"             // время загрузки
}
----
