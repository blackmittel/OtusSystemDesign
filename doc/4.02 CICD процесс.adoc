=== CI/CD процесс

==== Инфраструктура кода
* *Git* — хранение кода всех микросервисов и конфигураций
* *отдельные репозитории* для каждого сервиса
* Dockerfile в репозитории

==== Сборка
* Каждый push в ветку -> запуск сборки
* *Этапы:*
  1. *Сборка:* Docker-образ для каждого сервиса
  2. *Unit-тесты:* Проверка бизнес-логики

==== Автоматическое тестирование
* *Интеграционные тесты:* Проверка взаимодействия сервисов в тестовом окружении
* *E2E-тесты:* Критические сценарии (покупка товара)
* *Нагрузочные тесты:* Для основных API (поиск, создание заказа) на отдельном стенде
* *Security-тесты:* Проверка OWASP уязвимостей перед релизом

==== Развертывание
* *Blue-Green развертывание:*
. Развертывание новой версии параллельно со старой
. Направление части трафика на новую версию
. Мониторинг метрик и логов
. Постепенное увеличение трафика до 100%
* *Миграции БД:* Автоматическое применение через Liquibase
* *Канареечное развертывание* для критичных сервисов (платежи, заказы)

==== Окружения
. *IFT* — для разработчиков
. *Preprod* — полная копия production, тестирование интеграций
. *Production* — основное рабочее окружение с контролируемыми деплоями

==== Проверка развертывания
* *Проверка* сервисов после развертывания
* *Метрики* Контроль latency, error rate, RPS после обновления
* *Автоматический откат:* При обнаружении проблем в течение 5 минут
