===== Атрибутный контроль доступа (ABAC)

RBAC дополняется атрибутами для критичных операций.

====== Атрибуты
* пользователь
* роль
* владелец ресурса
* признак прохождения 2FA
* тип операции
* чувствительность данных
* контекст доступа (IP, география, сессия)

====== Правила

* Доступ к ресурсам:
* только владелец или привилегированная роль

* Заказы:
* доступ только участникам

* Финансовые операции:
* обязательная 2FA
* проверка владения

* Персональные данные:
* ограничение по чувствительности

* Административные операции:
* повышенные требования (роль + 2FA + ограниченный доступ)

---

==== Аутентификация и сессии

* OAuth2 / OpenID Connect (при необходимости)
* Локальная аутентификация (логин/пароль + 2FA)
* Сессии:
* завершение при неактивности (30 минут)
* refresh-токены с ограниченным сроком
* принудительное завершение

---

===== Двухфакторная аутентификация

Обязательна для:
* финансовых операций
* изменения чувствительных данных
* административных действий

Методы:
* OTP (SMS / push)
* TOTP

---

==== Управление секретами и ключами

===== Типы секретов
* JWT-ключи
* пароли пользователей
* API-ключи интеграций
* ключи шифрования данных
* TLS-сертификаты

===== Хранение
* централизованная система управления секретами
* ограниченный доступ по ролям
* аудит операций

===== Жизненный цикл
* генерация централизованно
* регулярная ротация
* отзыв при компрометации

---

==== Шифрование

* Данные в БД — AES-256 (на уровне приложения)
* Передача — TLS 1.2+
* Межсервисное взаимодействие — защищённые соединения (mTLS)

---

==== Реализация контроля доступа

* API Gateway:
* аутентификация
* rate limiting

* Сервисы:
* RBAC
* ABAC

* JWT:
* содержит роль и базовые атрибуты

---

==== Защита от атак

* DDoS — через CDN и внешние фильтры
* Rate limiting
* Защита от SQL/XSS
* Валидация входных данных
