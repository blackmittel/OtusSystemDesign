=== Схема взаимодействия

==== Принципы взаимодействия

===== Синхронные
- мгновенный ответ
- строгая консистентность
- короткие операции

Протоколы:
- HTTPS (REST)
- gRPC

===== Асинхронные
- устойчивость к сбоям
- fan-out
- eventual consistency

Протокол:
- Kafka

===== Messaging модель

- отправка сообщения: синхронный write
- получение сообщений: polling
- Kafka НЕ участвует в доставке сообщений пользователю
- Kafka используется только для событий (analytics, notifications)
