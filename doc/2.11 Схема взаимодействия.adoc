=== Схема взаимодействия

==== Принципы взаимодействия

===== Синхронные
- мгновенный ответ
- строгая консистентность
- короткие операции

Протоколы:
- HTTPS (REST)
- gRPC

===== Асинхронные
- устойчивость к сбоям
- fan-out
- eventual consistency

Протокол:
- Kafka

===== Внешние асинхронные взаимодействия (Webhook)

Используются когда:
- внешний провайдер инициирует событие
- требуется near real-time уведомление
- источник истины находится вне системы

Протокол:
- HTTPS (webhook)

===== Messaging модель

- отправка сообщения: синхронный write (Client → API → Ticket Service)
- получение сообщений: polling (Client → API → Ticket Service)
- Kafka НЕ участвует в доставке сообщений пользователю
- Kafka используется только для событий (analytics, notifications)
